Push External Project to Git
    
        
  • Open project in editor
    • 
        
  • In terminal type git init
    • 
        
  • git status
    • 
        
  • git add. or git add filename
    • 
        
  • git commit -m "--commit mssage--"
    • 
        
  • Now create a Repository in GitHub
    • 
        
  • Copy the html code of the Repository
    • 
        
  • In terminal write git remote add origin copiedHTMLCode
    • 
        
  • Check remote git  by typing git remote -v
    • 
        
  • Finish by typing git push -u origin master

      To add new featues , fix bug(s)and make change(s) the code without affecting the original code 
      concept of branching is used. till the time we won't merge the branch to the master branch the 
      changes will not be seen in the master branch/original code.
    
      
  • to check at which branch you are : git branch
    • 
      
  • To create a branch : git checkout -b nameOfTheBranch
    • 
      
  • To switch between branches : git checkout nameOfTheBranch
    • 
      
  • To compare the branch code with master code : git diff
    • 
      

      Once the brance is commited we will push the branch code.After all this we have to pull the branch
      to the master this we will do in Git scince it as automatically get to know that we have created 
      a branch,from there only we will merge the branch to the mastr branch.
      It may happen that many developer are woking on branch and trying to merge to the same master
      branch which may create a conflict. Now to resolve the conflict first commit properly then while 
      merging tell the git whether to keep both the changes or one particular or none after that merge 
      the branch this way both the branches will be merge.
